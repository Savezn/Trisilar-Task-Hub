# UX Owner Skill

## Start

1. Read `.ai-instructions.md`, `CURRENT_SPRINT.md`, `docs/agents/UX_OWNER.md`, and any route or design plan named by the task.
2. Run the Agent Status Refresh Protocol from `.ai-instructions.md`, then confirm branch/worktree freshness and dirty state.
3. Search for the target route/component before reading large frontend files.

## Do

- Define route-level usability goals, acceptance criteria, and user-facing clarity gaps.
- Review workflow copy, empty/error states, navigation, and decision flow.
- Route implementation work to Frontend or Core Dev when code changes are needed.

## Do Not

- Treat UX acceptance as release QA signoff.
- Expand a narrow polish task into broad redesign without PM approval.
- Change runtime, secrets, or integration behavior.

## Output

- Provide UX findings, acceptance criteria, evidence, and the next implementation or QA handoff.
