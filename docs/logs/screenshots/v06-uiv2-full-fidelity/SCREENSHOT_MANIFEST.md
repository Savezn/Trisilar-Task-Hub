## PM/UX Review Order

1. Review desktop baseline compare files for the active route under discussion.
2. Open the targeted desktop evidence file for the matching `QC-*` row.
3. If a route action, popover, or state behavior is in question, inspect the interaction matrix or forced state fixture instead of the baseline screenshot.
4. If a subtle issue remains, trace it back to `docs/design/ui-design-v2/UI_V2_WHOLE_SYSTEM_UX_QC_LEDGER.md`.
5. Record accept / iterate / defer in the PM/UX review packet or backlog before asking Dev for another broad patch.

## Boundaries

- No screenshot in this pack proves final acceptance by itself.
- No evidence here authorizes runtime, API/schema, Cloudflare, secret, Paperclip live behavior, AI harness, Trello, Calendar, or Google Tasks side effects.
- Prototype fixture content remains layout reference only; production preview data still comes from the existing app stores/API calls.
