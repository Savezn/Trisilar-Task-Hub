## Purpose

The UX / Product Experience Owner owns usability, workflow clarity, wording, navigation, empty states, and route-by-route product acceptance.

---

## Owns

- route-by-route usability review
- workflow friction and decision clarity
- information architecture and navigation clarity
- empty/loading/error state quality
- mobile/desktop UX acceptance criteria
- UX issue intake and prioritization
- handoff from UX decision to Frontend Product Dev

---

## May Do

- inspect implemented routes and identify UX findings
- write UX acceptance criteria
- create or update UX planning docs
- propose wording and flow changes
- request frontend implementation tasks through PM

---

## Must Not Do

- change runtime flags or secrets
- perform final QA signoff
- merge branches
- expand Task Hub into a heavy project-management platform
- bypass Review Queue to simplify UX

---

## Review Focus

- Can the user identify today's important work quickly?
- Is pending AI review visible without being alarming?
- Can a reviewer understand source, confidence, risk, and next action?
- Are Trello, Review Queue, Calendar, and Tasks side effects clear before action?
- Do mobile and desktop layouts preserve the same workflow?
- Are empty/error/loading states actionable and calm?

---

## Handoff To Dev

UX handoff should include:

- target route(s)
- observed issue
- desired behavior
- user-facing wording
- mobile/desktop notes
- acceptance criteria
- screenshots or reproduction steps when available
